# Архитектура проекта

## Общая модель

Проект разделён на два слоя:

- `theme` отвечает за визуальную часть, шаблоны, стили, SEO-разметку и WooCommerce presentation layer
- `plugin` отвечает за контентные сущности, админку, метабоксы, формы и проектную бизнес-логику

Это позволяет:

- не смешивать дизайн и проектную логику
- удобнее поддерживать сайт в будущем
- переносить тему и плагин по отдельности
- адаптировать проект под другого клиента без полного переписывания

## Структура

### Тема

Путь:

- `wp-content/themes/matreshka-master`

Отвечает за:

- общий layout
- главную страницу
- шаблоны контентных страниц
- WooCommerce шаблон магазина
- кастомные стили и JS
- fallback-навигацию
- schema / helper-слой

Ключевые файлы:

- `functions.php`
- `woocommerce.php`
- `template-parts/homepage.php`
- `assets/css/main.css`
- `assets/js/main.js`
- `inc/theme-setup.php`
- `inc/helpers.php`
- `inc/assets.php`
- `inc/schema.php`

### Плагин

Путь:

- `wp-content/plugins/matreshka-master-core`

Отвечает за:

- CPT `mm_showcase`
- CPT `mm_faq`
- CPT `mm_lead`
- метабоксы главной страницы
- глобальные настройки проекта
- обработку форм
- интеграционные точки
- сидирование demo-данных

Ключевые файлы:

- `matreshka-master-core.php`
- `includes/class-mm-admin.php`
- `includes/class-mm-forms.php`
- `includes/class-mm-content.php`
- `includes/class-mm-helpers.php`

## Контентные сущности

### Homepage

Главная страница управляется через обычную страницу WordPress с шаблоном `Homepage Builder` и набором метабоксов.

Редактируемые зоны:

- Hero
- видимость секций
- тексты витрин
- блок «Для элиты»
- блок о мастерской
- контактный блок

### Showcases

Витрины вынесены в отдельный тип записей. Это позволяет:

- хранить карточки отдельно от страницы
- легко менять порядок
- разделять карточки по категориям витрин

Категории:

- `prints`
- `corporate`
- `portrait`

### FAQ

FAQ реализован как отдельный тип записей, чтобы владелец сайта редактировал вопросы и ответы без захода в код.

### Leads

Заявки с форм сохраняются в админке для контроля и последующей интеграции с CRM.

## WooCommerce

WooCommerce используется как основа каталога.

Реализовано:

- страница магазина
- карточки товаров
- страница товара
- стилизация каталога и single product
- демо-товары для презентации

Важно:

Для полноценного мультиязычного WooCommerce-каталога нужен production add-on:

- `Polylang for WooCommerce`
- или `WPML + WooCommerce Multilingual`

## Мультиязычность

Текущая архитектура уже допускает три языка:

- русский
- английский
- китайский

Что подготовлено:

- языковые версии страниц
- переключатель языка
- структура переводимых зон
- возможность масштабировать систему на новые языки

Что требует production-настройки:

- переводы каталога WooCommerce
- перевод таксономий товаров
- единая multilingual-логика корзины и checkout

## Формы

Формы реализованы в проектном плагине.

Сценарии:

- заказать проект
- узнать стоимость по фото
- корпоративное предложение
- личный менеджер

Подготовлены точки подключения для:

- Bitrix
- Telegram
- WhatsApp
- MAX

## Почему такая архитектура выбрана

- WordPress остаётся удобной CMS для владельца
- WooCommerce закрывает каталог и оплату
- кастомная тема сохраняет премиальный UI без лишнего перегруза
- custom plugin делает систему поддерживаемой и переносимой
- проект легко расширять без переделки всего сайта
